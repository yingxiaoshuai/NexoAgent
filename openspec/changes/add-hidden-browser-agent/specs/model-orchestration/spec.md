## MODIFIED Requirements

### Requirement: 精简编排器工具面
编排器 SHALL 在精简的内置工具目录下运行，应当在交互式 Web 浏览和 Web 应用操作任务中使用 `browser_action`，且不得在默认运行时规划中依赖已移除的专用工具。

#### Scenario: 编排器从精简工具集中选择
- **当** 运行时准备编排器提示词和工具绑定时
- **则** 可用工具面应当排除已移除的文件、HTTP、技能和定时任务工具
- **并且** 允许范围收窄的 `browser_action`

#### Scenario: 编排器使用 shell 命令执行操作工作
- **当** 用户请求文件系统或命令行任务，且不需要多模态、记忆、知识库或浏览器行为时
- **则** 编排器应当优先使用 `shell_command` 而非任何已移除的专用工具路径

#### Scenario: 编排器使用浏览器动作操作 Web 应用
- **当** 用户请求 Agent 导航网页、检查网站或与 Web 应用中的控件交互时
- **则** 编排器应当在需要浏览器交互时使用 `browser_action`
- **并且** 不应当使用 `shell_command` 或已移除的 HTTP/Web 搜索工具路径来替代浏览器交互

#### Scenario: 编排器选择正确的浏览器 action
- **当** 编排器需要读取页面状态、导航、点击、输入、滚动、截图、刷新、后退或前进时
- **则** 它应当通过 `browser_action.action` 选择对应动作
- **并且** 在引用失效后应当重新请求 `snapshot`
