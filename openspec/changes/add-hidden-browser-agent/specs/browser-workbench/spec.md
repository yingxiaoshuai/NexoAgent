## ADDED Requirements

### Requirement: 浏览器工作台入口
系统 SHALL 提供一个用户可见的浏览器工作台入口，使用户可以从主应用菜单或侧边栏启动类似浏览器的功能。

#### Scenario: 从主导航打开浏览器工作台
- **当** 用户点击主应用中的浏览器入口时
- **则** 系统应当显示浏览器工作台视图
- **并且** 不应当离开或关闭 Nexo Agent 主应用

#### Scenario: 浏览器入口具备本地化文案
- **当** 用户界面语言为中文或英文时
- **则** 浏览器入口和核心控件应当显示对应语言的文案或可理解的图标提示

### Requirement: 浏览器式导航 Header
浏览器工作台 SHALL 在顶部显示地址栏和常用导航控件。

#### Scenario: 地址栏显示当前 URL
- **当** 浏览器工作台加载或页面导航完成时
- **则** 地址栏应当显示当前页面 URL

#### Scenario: 用户从地址栏导航
- **当** 用户在地址栏输入 URL 并提交时
- **则** 系统应当在工作台左侧浏览器区域加载该 URL
- **并且** 地址栏应当在导航完成后同步为实际 URL

#### Scenario: 刷新当前页面
- **当** 用户点击刷新控件时
- **则** 系统应当刷新当前浏览器页面

#### Scenario: 前进和后退
- **当** 用户点击后退或前进控件时
- **则** 系统应当在当前浏览器历史中后退或前进
- **并且** 当对应历史不可用时控件应当禁用或显示不可用状态

### Requirement: 左右分栏浏览器布局
浏览器工作台主体 SHALL 分为左侧浏览器内容区域和右侧 AI 聊天控制区域，并允许用户调整两者宽度。

#### Scenario: 左侧显示浏览器内容
- **当** 浏览器工作台处于打开状态时
- **则** 左侧主区域应当显示共享 Electron 浏览器会话中的真实 Web 页面内容

#### Scenario: 右侧显示 AI 控制聊天
- **当** 浏览器工作台处于打开状态时
- **则** 右侧区域应当显示 AI 聊天控制面板
- **并且** 用户在右侧发送的浏览器操作请求应当能够控制左侧浏览器页面

#### Scenario: 用户拖拽调整分栏大小
- **当** 用户拖拽左右分栏之间的分隔条时
- **则** 系统应当调整浏览器内容区域和 AI 聊天区域的宽度
- **并且** 调整过程中不应当导致 Header、页面内容或聊天输入区域互相重叠

#### Scenario: 记住分栏大小
- **当** 用户调整过分栏大小并重新打开浏览器工作台时
- **则** 系统应当恢复最近保存的分栏宽度偏好

### Requirement: AI 控制共享浏览器会话
浏览器工作台右侧 AI 聊天 SHALL 通过共享浏览器会话控制左侧页面，而不是创建独立的不可见页面。

#### Scenario: AI 操作影响左侧页面
- **当** 用户在右侧 AI 聊天中请求点击、输入、滚动或导航当前页面时
- **则** Agent 应当通过 `browser_action` 操作共享浏览器会话
- **并且** 左侧浏览器内容应当反映操作后的页面状态

#### Scenario: 手动页面状态可供 AI 读取
- **当** 用户在左侧浏览器区域手动导航或登录后
- **则** 后续 `browser_action` 的 `snapshot` 应当读取同一页面状态

#### Scenario: 离开工作台保留会话
- **当** 用户切换到其他 Nexo Agent 视图后再返回浏览器工作台时
- **则** 浏览器工作台应当保留共享会话的当前页面或可恢复最近状态

### Requirement: 工作台安全隔离
浏览器工作台 SHALL 将外部 Web 内容与 Nexo Agent 应用 UI 和本地权限隔离。

#### Scenario: 外部页面无 Node 权限
- **当** 左侧浏览器区域加载外部 Web 页面时
- **则** 外部页面不应当获得 Node integration 或 Nexo preload API 权限

#### Scenario: 拦截不受控的新窗口
- **当** 外部页面尝试打开新窗口或导航到非 Web 协议时
- **则** 系统应当按安全策略拦截、拒绝或交给系统浏览器处理
